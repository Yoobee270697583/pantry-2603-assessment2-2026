### "Shopping Lists"
- Auto-generated by comparing **planned meals** requirements against **pantry inventory** — any items the user doesn't already own/have are added to the shopping list
- Calculates deltas (e.g. recipe needs 400g pasta, pantry has 200g → list shows 200g)
- **Per-item controls** — each item on the list has:
  - **"Add to pantry"** — for when the user already owns the ingredient but hasn't updated their pantry inventory yet (moves/registers it into pantry)
  - **Quantity adjust** — increase or decrease the quantity being ordered
  - **Delete** — remove the item from the shopping list
- **Manual add** — the shopping list screen has its own "add item" button that opens a dialogue to search for ingredients independently of planned meals (e.g. snacks, beverages, alcohol, other staples)
- **"Add List To Pantry" button** (on the Shopping Lists screen) — the single key action that, in one tap:
  1. Updates the Pantry inventory with all items on the shopping list
  2. Clears/resets the shopping list screen
  3. Generates an order entry under Previous Orders / Order History
- An order is **only** created via this button — there is no separate "place order" step.
